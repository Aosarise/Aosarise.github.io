多表概念

  • 主键，外键
  • 主从表：主表有从表的键作为外键
  • 一对一：外键唯一
  • 一对多：外键不唯一。（包含外键的叫做多方）
  • 多对多：两个一方，通过多方相连。（包含外键的叫做多方）
    也就是需要一个中间表。

三种连接

  • 交叉连接 cross join
    • 隐式：select * from a,b –只是合并两张表的记录 没有关系可言
    • 显式：select * from a cross join b –笛卡尔积
  • 内连接 inner join 即：等值连接
    • 隐式：select * from a,b where a.id = b.id //这个写法默认作为隐式连接
    • 显式：select * from a inner join b on a.id = b.id
  • 外连接 分为左 右 全
    • 左：select * from a left join left b on a.id = b.id。另外两个类似写法
    • 外连接需要主表（保留表）的概念，左外连接即左表为主表。保留表所有的记录都保留，加上从表的列，如果从表没有记录则补NULL。
    • 全外连接（mysql不支持）full join

分页

  • 逻辑分页：将数据库的数据查询到内存中再进行分页
  • 物理分页：通过limit关键字，直接在数据库中进行分页，最终返回的数据是分页后的数据。
    mysql关键字: limit

子查询 不建议用

允许把一个查询嵌套在另一个查询里。
子查询，也叫做内部查询，包含内部查询的就叫做外部查询。
子查询可以用含普通的select可以包含的任何子句，如join、limit
外部查询必须是一下语句之一：select、insert、update、delete

位置：

  • select 中：只能返回一条.
  • from后
  • where中: 比如 id in (select pid from…)。或者 id = (select pid from…) //=的话只能返回唯一值
  • group by , order by 可用，但没啥意义 逻辑上来说就是返回一个表，或者一个键值对。我目前还不知道有什么使用场景。
