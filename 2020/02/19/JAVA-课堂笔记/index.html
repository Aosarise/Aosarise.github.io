Something before star

主要记录一下比较容易混淆的知识，还有一些语法上的细节。后面学习swing的时候可能才要认真记录。
记录一下我认为是重点的：传值

继承

  • 子类执行构造函数会先调用父类的构造函数

多态

  • 静态多态：同类方法，签名不同。签名：名称，参数类型，参数顺序。
  • 动态多态：重写父类方法。父类对象调用方法也是子类的方法。

静态

静态方法只能调用同一个类的静态字段，比如其他静态方法，成员。

final

常量。

接口

成员方法 默认abstract，成员域。
用来规定一个行为。

增强for

优点：不用判断边界，好写。
缺点：

  • 只能顺次遍历所有元素。不能访问下表。
  • 对于集合，就不方便删除集合内容

新switch

java7之前只能用基本类型和其封装类型。
现在可以用string类型

传值

java中所有的实参都是按值传递。
方法调用：

  • 基本类型的值的副本（改动不影响）
  • 对象引用的副本（指向同一个对象）

游长实参表

一个参数表只能出现一次省略号，并且只能放在参数表的末尾。
传入的实参表是一个数组。

this引用

没个对象都可以用this引用自己。
this调用同一个类的构造函数，只能放在第一行。

参数化类型，允许创建一些类，接口，方法操作的数据类型被指定为一个参数。
这种类型参数可以在类、接口、方法的创建中，分别为泛型类、泛型接口、泛型方法。
优点：类型安全，代码重用率
所有泛型类都会用Object(上界)替换.

文件 流

流：信息的逻辑实体。指从数据源读取，或者写入目标对象的有序数据。
流分类：输入、输出 ; 字节流(基础单元为8bit的字节)、字符流(16bit) ;节点流(面向特定的io设备，也为低级流)、处理流(对已存在的流进行包装)
Java将每个文件都视为流的存储。文件本身没有”记录”的概念。
基于字节的文件：二进制文件，基于值编码
基于字符：文本文件，基于ascII码，或者unicode。
差别在于编码形式。物理层是一样的

序列化

对象序列化(Serialize)是指将一个Java对象写入IO流中
 对象反序列化(Deserialize)是指从IO流中恢复该Java对象
 序列化必须实现Serializable和Externalizable接口
场景：通过socket进行对象的复制和通讯等、 程序版本升级问题
Serializable接口是一个标记接口，这样的接口不包含方法
