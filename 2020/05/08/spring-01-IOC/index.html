Spring介绍

Spring的核心就是IOC，即Spring来进行很多配置的流程，以及资源的管理。IOC即控制反转，由Spring框架来控制资源。
在Spring里，一个实例称为一个bean，bean的创建和销毁都由Spring管理。同时也可以通过Spring轻松实现代理模式，管理方法的运行

IOC容器

是什么

Spring中Bean工厂里的Map存储机构。
Bean工厂主要接口:ApplicationContext。常用的一些上写文都是implement这个接口的。
最原始的底层接口为: BeanFactory接口。//可以自己实现这个接口来使用bean对象。
存储机构：DefaultListableBeanFactory。//这便是具体存放数据的地方

创建

首先介绍几个ApplicationContext子类

  1. ClassPathXmlApplicationContext：从跟路径下加载配置文件
  2. FileSystemXmlApplicationContext: 从磁盘路径加载文件
  3. WebApplicationContext: web容器，一般是配置web.xml的监听器时自动创建

具体创建：
ApplicationContext context = new ClassPathXmlApplicationContext(xml路径);

DI依赖注入_XML

DI注入的意思是，IOC创建的实例有时候需要参数，DI就是用来注入参数的。
参数分成几种类型：

  1. 简单类型: 字符串，数字，bool。
  2. POJO类型：用户自定义的类。注入时候需要先包装成bean，再注入
  3. 集合类型：主要指的是set，List，Map

注入的方法有两种//不用直接调用，不过配置bean的时候写法不同，等下就能看到

  1. 构造函数
  2. setter函数

@Resource @Autowired @Qualifier

这个就是POJO类型的依赖注入。这三个注解有所不同

  1. @Resource: 默认按照名称(id)进行装配，可以通过name 来指定id，缺省则为字段名。定义了name的状态下只能通过名称装配，缺省状态下找不到匹配名称就按照类型装配。不属于Spring，属于J2EE。大概用这个降耦合度，不过我暂时感受不到差别。
  2. @Autowired:按照类型装配。如果想使用名称装配可以结合@Qualifier注解进行使用
  3. @Qualifier:在自动按照类型注入的基础之上，再按照 Bean 的 id 注入。它在给字段注入时不能独立使用，必须和@Autowire 一起使用；但是给方法参数注入时，可以独立使用。用的不多我也不会
